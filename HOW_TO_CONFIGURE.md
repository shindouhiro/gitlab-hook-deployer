# GitLab 自动部署配置指南 🚀

本文档将指导您如何通过 `http://localhost:5173` 的新版界面配置 GitLab Webhook，实现多分支自动化部署。

## 1. 运行环境准备

在开始配置前，请确保后端和前端均已启动：

- **后端 (FastAPI)**: 在根目录运行 `uv run uvicorn app.main:app --reload --port 8000`
- **前端 (Vue 3)**: 在 `web/apps/ui` 目录运行 `pnpm dev` (访问 `http://localhost:5173`)

---

## 2. 界面配置步骤

打开浏览器进入控制台界面，按照以下步骤操作：

### 第一步：选择项目
1. 点击左侧“Webhook 配置”面板中的 **“刷新”** 按钮。
2. 在下拉列表中选择您想要配置自动部署的 GitLab 项目。

### 第二步：设置 Hook URL
- 界面会自动填充默认地址（通常是您的后端 Webhook 接收端）。
- **注意**：如果您的代码运行在本地且 GitLab 在公网（如 gitlab.com），您需要使用 `ngrok` 或 `frp` 将本地 8000 端口映射到公网，并将公网地址填入此处。

### 第三步：配置触发分支（核心功能）
- **任何分支 (Any Branch)**：只要有代码推送，无论哪个分支都会触发部署。
- **指定分支 (Specific Branches)**：
    1. 选中该项。
    2. 在输入框内输入分支名（例如 `main`），按 **Enter (回车)**。
    3. 您可以添加多个分支标签（例如 `test`, `release/*`），系统会自动处理匹配逻辑。

### 第四步：设置安全 Token (可选)
- 输入一个自定义字符串作为 Secret Token，用于验证 GitLab 回调的合法性。

### 第五步：一键应用
- 点击 **“一键应用配置”**。
- 此时系统会自动调用 GitLab API，在您的项目设置中创建一个 Webhook，并勾选 `Push events` 触发器。

---

## 3. 触发部署与实时监控

配置完成后，自动化流程如下：

1. **推送代码**：在您的 GitLab 项目中执行 `git push`。
2. **接收回调**：GitLab 会向您的后端发送 Webhook 请求。
3. **获取 Task ID**：
    - 如果您有配置钉钉或飞书通知，后端会自动推送包含 `task_id` 的消息。
    - 或者您可以查阅后端日志获取最新的 `task_id`。
4. **实时监控**：
    - 将 `task_id` 粘贴到界面右侧的“任务 ID 监听流”输入框。
    - 点击 **“连接实时监控”**。
    - 界面会实时展示：**拉取代码 -> 构建 -> 发布 -> 健康检查 -> 部署** 的全过程及详细日志。

---

## 💡 技术贴士
- **多分支过滤机制**：由于 GitLab 原生 Webhook 仅支持单个通配符，我们通过在 Webhook URL 后追加 `?branches=main,test` 参数，由后端服务在接收到请求时进行二次校验，实现了灵活的多分支触发支持。
- **UnoCSS 样式**：界面基于 UnoCSS 构建，具有极速的响应速度和现代化的视觉效果。
